1. Architecture overview

Three trust zones: the browser (untrusted), Firebase (Google-managed identity provider), and the SERVI backend on Render (authoritative for app data). Postgres on Neon stores user records and addresses. The browser holds two distinct tokens at once.

2.4 · Session establishment (POST /api/auth/firebase)

Common terminator for all three signup paths above. Issues the SERVI custom JWT.

sequenceDiagram
  autonumber
  participant FE as Browser
  participant BE as Backend
  participant FB as Firebase Admin
  participant PG as Postgres

  FE->>BE: POST {firebase_id_token, name?, email?, phone?}
  BE->>BE: rate-limit (publicFormLimit · 5/min/IP)
  BE->>FB: verifyIdToken(idToken, checkRevoked=true)
  FB-->>BE: {uid, email_verified, phone_number, sign_in_provider}
  BE->>PG: lookup by firebase_uid → email → phone
  alt Found existing
    BE->>PG: UPDATE auth_users SET ...
  else Cross-identifier orphan
    BE->>PG: MERGE phone-only into firebase_uid
  else New
    BE->>PG: INSERT auth_users
  end
  BE->>BE: signSessionToken({user_id, email, phone, ...})
  BE-->>FE: 200 {token, user}

2.5 · Booking gate

Step 3 of the service-request flow requires both email_verified=true AND phone_verified=true.

sequenceDiagram
  autonumber
  participant FE as Browser
(service.html)
  participant BE as Backend
  participant PG as Postgres

  FE->>BE: POST /api/service-requests
Authorization: Bearer JWT
  BE->>BE: verifySessionToken(JWT)
  BE->>PG: SELECT email_verified, phone_verified
  alt Both true
    BE->>PG: INSERT service_requests
    BE-->>FE: 200 {request_id}
  else Either false
    BE-->>FE: 403 {needs: ['email','phone']}
    FE->>FE: Show verification modal
  end

2.6 · Profile edit (PATCH /api/auth/me)

Currently runs requireRecentAuth (Firebase token ≤ 5min old) before applying changes. Phone change does NOT reset phone_verified A4.

sequenceDiagram
  autonumber
  participant FE as Browser
(account.html)
  participant BE as Backend
  participant FB as Firebase Admin
  participant PG as Postgres

  FE->>BE: PATCH /api/auth/me
{name, email?, phone?}
X-Firebase-Token (fresh)
  BE->>BE: verifySessionToken(JWT)
  BE->>FB: verifyIdToken(firebaseToken, checkRevoked=true)
  FB-->>BE: decoded {auth_time}
  BE->>BE: requireRecentAuth (auth_time within 300s) A7
  alt email changed
    BE->>PG: check uniqueness
    BE->>PG: UPDATE auth_users (email_verified left as-is)
  end
  alt phone changed
    Note over BE,PG: ⚠ phone_verified NOT reset (A4)
    BE->>PG: UPDATE auth_users SET phone=$new
  end
  BE-->>FE: 200 {user}
